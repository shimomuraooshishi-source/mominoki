# 要件定義書

**プロジェクト名：** Gmail・日報・売上集計 照合管理ツール（MVP）  
**対象クライアント：** 株式会社みどリむし  
**作成日：** 2026-05-27  
**ステータス：** ドラフト v0.1

---

## 1. 背景・目的

毎日届く Gmail の施術通知・決済メールと、日報スプレッドシート・売上集計スプレッドシートを手動で突き合わせている。目視確認・手入力によるミスと工数を削減するため、3つのデータを自動で照合して差異を可視化する Web ツールを構築する。

---

## 2. MVP スコープ

### 含むもの

| # | 機能 | 概要 |
|---|------|------|
| 1 | Gmail 読み取り | OAuth 認証で施術通知・決済メールを自動取得 |
| 2 | スプレッドシート読み取り | 日報・売上集計シートを自動取得 |
| 3 | 照合・差異表示 | 3つのデータを突き合わせ、差異をハイライト表示 |
| 4 | 確認促し表示 | 差異があった場合に Airレジ・PayPay・楽天ペイの確認を促すメッセージ表示 |

### 含まないもの（MVP では対象外）

- Airレジ・PayPay・楽天ペイとのリアルタイム API 連携
- ユーザー権限管理・ロール分け
- モバイル対応
- 通知機能（メール・Slack 等）
- 過去データの統計・分析

## 4. 機能要件

### 4.1 Gmail 読み取り

- Google OAuth 2.0 で Gmail に接続する
- 対象メール条件（**要確認**）：
  - 差出人ドメイン・件名のパターン → TBD
  - 取得期間：当日分をデフォルト表示、日付指定で過去分も閲覧可
- 取得する情報（**要確認**）：
  - 施術日時、金額、決済方法、顧客識別子 → TBD（メール本文のパターン次第）

### 4.2 スプレッドシート読み取り

- Google Sheets API で日報・売上集計シートを取得する
- **日報シート**（**要確認**）：
  - スプレッドシート URL / シート名 → TBD
  - 照合に使う列（日付・施術者・顧客・施術内容など） → TBD
- **売上集計シート**（**要確認**）：
  - スプレッドシート URL / シート名 → TBD
  - 照合に使う列（日付・金額・決済方法など） → TBD

### 4.3 照合ロジック

- Gmail・日報・売上集計の3つを日付単位で突き合わせる
- 差異の種類：
  - Gmail にあるが日報にない（施術記録漏れ）
  - Gmail にあるが売上集計にない（売上記録漏れ）
  - 金額の不一致
  - 決済方法の不一致

### 4.4 差異表示画面

- 日付を選択して照合結果を表示する
- 差異がある行を色分けで強調表示する（例：赤＝不一致、黄＝片方のみ）
- 差異の種類ごとにフィルタリングできる
- 差異がある行に「Airレジを確認」「PayPayを確認」「楽天ペイを確認」のラベルを表示する

### 4.5 手動照合補助

- 差異行に対してスタッフがコメント・確認済みフラグを付けられる（MVP では簡易実装）

## 8. MVP 完成の定義

以下がすべて動作すれば MVP 完成とする。

- [ ] Google アカウントでログインできる
- [ ] 日付を指定すると Gmail から対象メールを取得できる
- [ ] 日報・売上集計スプレッドシートのデータを取得できる
- [ ] 3つのデータの照合結果が画面に表示される
- [ ] 差異がある行がハイライトされ、どの決済サービスを確認すべきか表示される
