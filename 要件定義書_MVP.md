**対象会社：** 株式会社みどリむし  
**ドキュメント版数：** v0.1（要件定義中）  
**作成日：** 2026-05-27  

---

## 1. プロジェクト概要

### 背景と目的

現在、施術の実績確認は「Gmail のメールを見ながら日報・売上集計スプレッドシートと手動で照合する」作業で行われている。この作業は時間がかかるうえ、目視による確認ミス・入力漏れのリスクがある。

本ツールは Gmail・日報・売上集計の 3 データを自動で照合し、差異を一覧表示することで、**照合作業の時間削減とミスの防止**を目的とする。

### ツール概要

| 項目 | 内容 |
|------|------|
| ツール形式 | PC ブラウザで動作する Web アプリ |
| 利用者 | 社内チーム（複数人） |
| データソース | Gmail・Google スプレッドシート（日報・売上集計） |

---

## 2. MVP スコープ

### 対象機能

| ID | 機能名 | 概要 |
|----|--------|------|
| F1 | Gmail 自動読み取り | 施術通知・決済メールを Gmail API で取得し、日付・金額・施術者などの情報を構造化する |
| F2 | 日報との照合 | Gmail から取得した施術記録と、日報スプレッドシートの記録を日付・施術者単位で突き合わせる |
| F3 | 売上集計との照合 | Gmail から取得した決済金額と、売上集計スプレッドシートの金額を突き合わせる |
| F4 | 差異の一覧表示 | 3 データ間のズレを日付・項目別に一覧表示し、差異がある行をハイライトする |

### スコープ外（MVP では作らない）

- Airレジ・PayPay・楽天ペイとの自動連携（差異発見時の手動確認は対象内）
- ユーザー権限管理・ログイン機能
- モバイル対応
- 過去データの統計・グラフ表示

## 4. 機能要件詳細

### F1：Gmail 自動読み取り

- Gmail API を使用して、指定条件のメールを自動取得する
- 取得対象：施術通知メール・決済完了メール
- 取得する情報：日付、施術者名または顧客名、金額、決済手段（PayPay / 楽天ペイ / Airレジ 等）
- 取得済みメールは重複取得しない

> 【要確認】差出人アドレス・件名のパターン（例：`件名に "予約確定" を含む` など）

### F2：日報との照合

- 日報スプレッドシートから当日分のデータを自動読み込む
- Gmail の施術記録と日報を、**照合キー**で突き合わせる
- 一致・不一致・片方のみ存在 の 3 状態を判定する

> 【要確認】  
> - 日報シートの列名と並び順（例：A列=日付、B列=施術者名、C列=施術内容…）  
> - 照合キーは何にするか（日付＋施術者名？ 予約番号？）

### F3：売上集計との照合

- 売上集計スプレッドシートから当日分の金額データを自動読み込む
- Gmail の決済金額と売上集計を日付単位で突き合わせる
- 金額の差分を計算して表示する

> 【要確認】  
> - 売上集計シートの列名と金額フォーマット（例：円表示か数値のみか）  
> - 決済手段ごとに別シートか、同一シートか

### F4：差異の一覧表示

- 照合結果を日付昇順で一覧表示する
- 差異がある行は色でハイライトする（例：赤＝不一致、黄＝片方のみ）
- 「差異のみ表示」フィルターを設ける
- 差異があった場合、確認すべき決済サービス（Airレジ・PayPay・楽天ペイ）を列に表示する

## 5. 非機能要件

| 項目 | 内容 |
|------|------|
| 動作環境 | PC ブラウザ（Chrome 最新版を基準） |
| データ更新 | 手動実行（ボタンクリックで照合を起動） |
| レスポンス | 照合処理は 30 秒以内 |
| セキュリティ | Gmail・スプレッドシートへのアクセスは OAuth2 認証を使用 |
